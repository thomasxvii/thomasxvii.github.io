Algorithms and Data Structure Enhancement

The Contact Service project, also from CS 320, originally implemented a linear ArrayList for managing contacts. The enhanced version replaces this with a HashMap (dictionary), which allows for constant-time lookup, simplifies logic for updating and deleting records, and ensures data integrity through duplicate prevention and improved validation. These changes significantly boost performance and reduce redundancy.

Through this enhancement, I demonstrated the impact of appropriate data structure selection and algorithmic improvements in real-world development scenarios. The narrative included outlines these modifications and how they met the course outcomes for performance and structure.

This enhancement refactors the Contact Service by replacing a list-based structure with a Python dictionary to improve lookup efficiency. This change simplifies the logic and ensures scalability for contact management features.

Databases Enhancement

The Grazioso Salvare project, originally developed in IT 145, was enhanced by transitioning from flat file-based storage to a normalized SQLite relational database. This not only provided persistent storage but also introduced secure parameterized queries and a modular code structure that improved both functionality and scalability.

By creating modules such as database.py, models.py, and main.py, I adhered to separation of concerns and clean architecture principles. These updates reflect my grasp of secure and scalable data design and the importance of integrating database systems into real-world applications. The supporting narrative outlines the database schema design, security considerations, and testing practices.

This project is a refactor of the Grazioso Salvare CLI app from IT 145. It now includes SQLite integration for persistent storage, structured queries, and normalized schema design for efficient data retrieval and security.
